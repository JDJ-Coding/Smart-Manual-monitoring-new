# 설치 가이드 — git clone 후 추가 설정

`git clone` 이후 아래 3가지를 수동으로 추가해야 정상 동작합니다.
(모두 `.gitignore`에 등록되어 있어 저장소에 포함되지 않습니다.)

---

## 수동 복사 필요 항목 요약

| # | 항목 | 방법 | 필수 여부 |
|---|------|------|-----------|
| 1 | `.env.local` | 직접 생성 | 필수 |
| 2 | `model/Xenova/bge-m3/` | USB 복사 (약 560MB) | 필수 |
| 3 | `data/manuals/` | PDF 파일 복사 | 필수 |

---

## 1. `.env.local` — 환경 변수 파일

프로젝트 루트에 `.env.local` 파일을 직접 생성합니다.

```env
# POSCO GPT API Key (필수)
POSCO_GPT_KEY=<사내_GPT_Bearer_토큰>

# 관리자 패널 비밀번호 (기본값: posco)
ADMIN_PASSWORD=posco

# 영어 PDF를 한국어로 번역 후 임베딩 (선택, DB 구축 시간 증가)
PDF_TRANSLATE=false

# 각 청크에 문맥 요약 추가 (선택, DB 구축 시간 대폭 증가)
CONTEXTUAL_RETRIEVAL=false
```

> `POSCO_GPT_KEY`는 Windows 시스템 환경 변수로 설정되어 있으면 `.env.local` 없이도 동작합니다.
> `ADMIN_PASSWORD` 기본값은 `posco`이며, 실 배포 시 변경을 권장합니다.

---

## 2. `model/` 폴더 — 임베딩 모델 (약 560MB)

로컬 ONNX 모델 파일이 담긴 폴더로, 인터넷 다운로드가 비활성화되어 있어 **USB로 복사**해야 합니다.

- 모델: `Xenova/bge-m3` (q8 quantized)
- 복사 위치: 프로젝트 루트의 `model/` 폴더

```
Smart-Manual-monitoring-new/
└── model/
    └── Xenova/
        └── bge-m3/       ← 이 폴더째 복사 (onnx/ 포함)
```

> `model/Xenova/bge-m3/onnx/` 하위 파일들이 모두 있어야 합니다.

---

## 3. `data/manuals/` — 설비 매뉴얼 PDF

PDF 파일들을 `data/manuals/` 폴더에 복사합니다.
폴더가 없으면 직접 생성하세요.

```
Smart-Manual-monitoring-new/
└── data/
    └── manuals/          ← PDF 파일들을 여기에 복사
        ├── FR-E800.pdf
        └── (기타 매뉴얼).pdf
```

> `data/vector-store/index.json` (벡터 DB)은 직접 옮길 필요 없이, 아래 설치 순서 6단계에서 자동 생성됩니다.

---

## 전체 설치 순서

```bash
# 1. 클론
git clone <repo-url>
cd Smart-Manual-monitoring-new

# 2. 의존성 설치
npm install

# 3. .env.local 생성 (위 내용 참고)

# 4. model/ 폴더를 USB에서 복사
#    → model/Xenova/bge-m3/ 폴더가 존재해야 함

# 5. data/manuals/ 에 PDF 복사

# 6. 개발 서버 실행
npm run dev
# → http://localhost:3000 접속 확인
```

서버 실행 후 **http://localhost:3000/admin** 에 접속하여 로그인(`posco`)한 뒤 **"DB 재구축"** 버튼을 눌러 벡터 DB를 생성합니다.

> 최초 DB 구축 시 임베딩 모델(BGE-M3) 초기화로 30~60초 소요될 수 있습니다.
